# Projetos
Essa página apresenta as principais informações acerca do sistema de projetos utilizado no repositório, bem como uma visão geral de uso eficiente 
dessa infraestrutura.

## O que são?
O sistema de projetos do GitHub é um método de reunir e organizar informações pertinentes a projetos específicos dentro da alçada de um mesmo repositório,
oferecendo uma forma conveniente de compartimentarizar os escopos das equipes, mas sem dissociar completamente as informações e os projetos de suas relações.

## Para que servem?
De forma prática, os projetos atuam como 'aglutinadores' de informação e planejamento, de modo a modularizar suficientemente o enfoque e a fronteira do
envolvimento das equipes individuais, mas sem deixar para trás as relações de cada grupo com o projeto maior - nesse caso, o LBTM.<br>
As informações de cada um desses grupos fica disponível para acesso de todos os outros, inclusive para o professor responsável pelo laboratório, permitindo
a colaboração de todos no processo da produção científica.

## Como desempenham sua função?
Cada projeto apresenta uma página com diferentes modos de visualização da informação ali contida:<br>
- Quadro
- Tabela
- Roadmap

Cada um desses pode ser utilizado de forma independente do outro, em qualquer quantidade, e atende a diferentes propósitos em relação ao outro.

#### Quadro
Os quadros tipicamente possuem colunas largas de visualização onde ficam posicionados [tickets](https://github.com/Neblinus/LBTM/tree/main/Guia%20de%20Uso/Tickets)
pertinentes ao status daquela coluna.<br>
Esses status parecem-se com as _labels_ explicadas no guia dos [tickets](https://github.com/Neblinus/LBTM/tree/main/Guia%20de%20Uso/Tickets), e assim como as
primeiras, são usados como um feedback visual rápido e ferramenta organizacional do estado de algo - no caso, a coluna em si é como um _placeholder_ para
tickets de uma mesma temática.
> [!NOTE]
> O ideal seria poder fixar discussões das seções de cada projeto ao seu respectivo quadro, mas como relatado [nessa issue](https://github.com/orgs/community/discussions/11223),
> o GitHub ainda não tem suporte a essa função.

#### Tabela
As tabelas são como uma lista de tickets padrão, mas que só é exibida na página do projeto.
> [!IMPORTANT]
> Ainda que as tabelas dos projetos só sejam exibidas na página do projeto, os tickets criados nelas são visíveis para todos na página geral de tickets.<br>
> Pode parecer contraditório, mas é vital para que mantenha-se a relação micro-macro.<br>
> A grande diferença é que, ao criar um ticket pela tabela de um projeto, ele é automaticamente associado ao projeto, facilitando a pesquisa por filtros.

Nessa lista, os tickets podem receber não só as _labels_ padrão, como em qualquer outro ticket, mas também, status específicos definidos pelo projeto.<br>
Isso permite indicar e visualizar rapidamente, por exemplo, quais ações devem ser prioridade no plano de execução de pesquisa do projeto, permitindo maior
controle e coesão do processo de produção científica.

#### Roadmap
As roadmaps são tabelas com um tipo de calendário integrado, permitindo que o projeto planeje e monitore o cumprimento de metas definidas num determinado
espaço de tempo.<br>
A interface para elas é menos agradável, mas ainda, utilizável. Elas também usam tickets como padrão de referência.
